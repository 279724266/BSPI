### 1 项目背景
- 智能安防需求增长
- 人脸识别技术成熟
- 树莓派硬件平台普及

### 2 项目目标
- 实现基于人脸识别的智能门禁系统
- 集成硬件控制与安全监测
- 提供用户友好的交互界面

### 3 技术栈
- 前端 ：HTML5 + CSS3 + JavaScript
- AI框架 ：TensorFlow.js + face-api.js
- 后端 ：Node.js + Express
- 硬件控制 ：rpio 库
- 通信协议 ：HTTP/RESTful API

## 二、技术架构
### 1 系统架构
- 前端识别 + 后端控制 + 硬件执行

### 2 数据流
- 摄像头 → 前端AI处理 → 后端API → 硬件控制

### 3 模块关系
- 人脸识别模块 ↔ 硬件控制模块 ↔ 安全监测模块

## 三、核心功能模块
### 1 人脸识别模块
- 实时人脸检测
- 人脸特征提取
- 身份验证对比
- 性别、年龄、表情识别
- 黑名单检测

### 2 硬件控制模块
- RGB LED控制（红/绿/蓝/白）
- 舵机控制（0°-180°）
- 硬件状态反馈

### 3 安全监测模块
- MQ-2烟雾/燃气传感器
- 烟雾检测与报警
- 短信推送通知

### 4 日志管理模块
- 识别日志记录
- 日志查询与过滤
- 日志导出（CSV）

### 5 黑名单管理模块
- 黑名单添加与删除
- 黑名单实时检测
- 黑名单警报触发

## 六、API接口
### 1 硬件控制接口
- `/led/:color` - LED控制
- `/servo/rotate/forward` - 舵机正向旋转
- `/servo/rotate/backward` - 舵机反向旋转
- `/servo/reset` - 舵机归位
- `/mq2/status` - MQ-2传感器状态

### 2 数据管理接口
- `/api/face-log` - 添加人脸识别日志
- `/api/face-logs` - 获取日志列表
- `/api/face-logs` (DELETE) - 清除日志
- `/api/blacklist` - 黑名单管理
- `/api/blacklist/:id` - 删除黑名单
